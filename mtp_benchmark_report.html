⚙️ MTP 配置说明

MTP (Multi-Token Prediction) 是 llama.cpp 的推测解码技术，通过一次前向传播预测多个 token 来加速推理。

启动命令 (启用 MTP): /home/jianzengliang/llama.cpp/build/bin/llama-server \ --model ./Qwen3.6-27B-Q4_K_M-mtp.gguf \ --port 8088 \ --ctx-size 8192 \ --n-gpu-layers -1 \ --spec-type draft-mtp \ --spec-draft-n-max 2 \ --flash-attn on 关键参数: • --spec-type draft-mtp # 使用 MTP 推测解码 • --spec-draft-n-max 2 # 最多推测 2 个 token

💡 测试结论与建议

✅ MTP 显著加速

MTP 在 Qwen3.6-27B 上实现了 1.46x 平均加速比，最高达到 1.63x（中长度提示词）。这说明 MTP 推测解码在该模型上非常有效。

📝 提示词长度影响

中长度提示词的加速比最高 (1.63x)，可能是因为：

  • 短提示词：受 TTFT 影响较大，MTP 优势不明显
  • 中提示词：MTP 推测命中率高，加速效果最佳
  • 长提示词：上下文较长，MTP 推测难度增加

⚠️ TTFT 变化

启用 MTP 后：

  • 短/中提示词：TTFT 降低 (243ms → 131ms, 602ms → 345ms)
  • 长提示词：TTFT 略增 (387ms → 431ms)，可能是因为 MTP 需要处理更多推测 token

🚀 进一步优化建议

当前 `--spec-draft-n-max 2` 只推测 2 个 token，可以尝试：

  • 增加推测 token 数：尝试 `--spec-draft-n-max 4` 或 `6`，可能获得更高加速比
  • 调整不接受惩罚：使用 `--spec-draft-penalty 0.0` 等参数优化推测策略
  • 批量测试：测试不同 `--spec-draft-n-max` 值 (2, 4, 6, 8) 找到最优配置

🎯 性能瓶颈分析

当前推理速度较低 (6-9 tok/s)，主要原因是：

  • 模型规模：27B 参数模型需要大量计算
  • 量化精度：Q4_K_M 量化在精度和速度间取得平衡
  • 内存带宽：DGX Spark 的 LPDDR5X (~273 GB/s) 是主要瓶颈
  • 如果升级到更高带宽内存（如 HBM3），速度可以进一步提升
