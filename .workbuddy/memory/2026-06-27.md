# 2026-06-27 工作日志

## 远程 DGX Spark ollama 性能测试

用户要求测试远程服务器 `jianzengliang@100.81.83.59` 上的三个 ollama 模型性能。

### 服务器信息
- SSH: `jianzengliang@100.81.83.59`
- Ollama API: `http://127.0.0.1:11434`（仅 localhost 可达）
- 测试方法：上传 `benchmark_ollama.py` 到 `/tmp/`，SSH 执行，`PYTHONUNBUFFERED=1` 解决输出缓冲问题

### 三个模型
| 模型 | 参数 | 量化 | 上下文 | 大小 |
|------|------|------|--------|------|
| `qwen3.6:35b` | 36B | Q4_K_M | 262K | 23 GB |
| `qwen3-coder-next:latest` | 79.7B | Q4_K_M | 262K | 51 GB |
| `glm-4.7-flash:latest` | 29.9B | Q4_K_M | 202K | 19 GB |

### 测试结果（平均 tok/s）

| 模型 | 短问答 | 代码生成 | 长文生成 | 综合平均 | 排名 |
|------|--------|----------|----------|----------|------|
| **qwen3.6:35b** | 74.4 | 74.8 | 74.1 | **74.4** | 🥇 |
| **glm-4.7-flash:latest** | 66.7 | 66.0 | 65.3 | **65.5** | 🥈 |
| **qwen3-coder-next:latest** | 49.0 | 49.8 | 50.2 | **49.7** | 🥉 |

### 关键发现
- `qwen3.6:35b` 最快且最稳定（三次运行几乎无波动，74~75 tok/s）
- `qwen3-coder-next` 首次运行含模型加载（51s），热启动后稳定在 50 tok/s
- `qwen3-coder-next` 短问答只生成 32 tokens（模型判断无需更多），吞吐量计算基于实际 token 数
- `glm-4.7-flash` 显存占用最小（19 GB），适合多模型并发

### 报告文件
- `benchmark_report.html` — 可视化报告（Chart.js 柱状图，深色主题）
- `benchmark_result.txt` — 原始文本结果
- `benchmark_ollama.py` — 测试脚本（纯 stdlib，无第三方依赖）

### 技术笔记
- SSH 远程执行时 `<>` 被 bash 当作重定向，需用 `python3 -c "..."` 方式避免
- Python stdout 缓冲导致无输出：用 `PYTHONUNBUFFERED=1` 或 `python3 -u`
- ollama API 的 `eval_duration` 字段（纳秒）才是真实推理时间，`total_time` 含网络/排队开销
