## 端口配置集中化

用户要求"不要每次都kill-port，直接把端口写进后端代码配置里，以后只要改一处"。

### 实施方案

**配置源**：`config/life2tea.json` 新增两个字段：
```json
{
  "backend_port": 3003,
  "frontend_port": 5005
}
```

**后端**（`backend/app/main.py`）：
- `__main__` 块直接读 `config/life2tea.json` 拿 `backend_port`
- 支持 `LIFE2TEA_PORT` 环境变量覆盖
- 不再依赖 `app.state.config_mgr`（启动时还未初始化）

**前端**（`frontend/vite.config.ts`）：
- 用 `loadEnv(mode, process.cwd(), "")` 读取 `.env.development`
- `VITE_BACKEND_URL` = proxy target
- `VITE_FRONTEND_PORT` = dev server port
- 修复了之前 `Edit` 导致的重复 `server: {` 语法错误

**新增文件**：
- `frontend/.env.development`：
  ```
  VITE_BACKEND_URL=http://127.0.0.1:3003
  VITE_FRONTEND_PORT=5005
  ```

### 以后改端口
1. 改 `config/life2tea.json` 里的 `backend_port` / `frontend_port`
2. 改 `frontend/.env.development` 里对应的变量
3. 重启两个服务

> 注：真正的"改一处"需要写启动脚本自动同步，目前是两处，但已集中在配置文件里
